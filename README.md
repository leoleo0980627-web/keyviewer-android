# KeyViewer for Android

KeyViewer 是一個 Android 上的鍵盤可視化工具，用來顯示實體鍵盤按鍵、統計 KPS / TOTAL、編輯按鍵外觀，並可把畫面以懸浮窗方式疊在其他 App 上。

目前專案主要依賴：

- `Shizuku` 讀取 `/dev/input/event*` 鍵盤事件
- 自訂 `KeyView` 編輯按鍵版面
- `FloatService + FloatView` 顯示懸浮窗
- `SharedPreferences` 與 JSON 匯入 / 匯出保存配置

## 目前狀態

### 已可用

- 實體鍵盤按鍵監聽與可視化
- KPS / TOTAL 顯示
- 懸浮窗顯示
- 按鍵位置、大小、文字、顏色、鍵雨效果編輯
- 圖層管理
- 配置匯入 / 匯出
- 深色 / 淺色切換

### 尚未完成或不應視為已可用

- 按鍵映射功能目前只有 UI 與部分資料結構，**不應視為已完整可用**
- 「不用 root 就能完整按鍵映射」這件事 **不符合目前程式現況**
- IME 服務目前主要仍是框架與入口，**不是完整輸入法映射實作**

## 版本更新

### v1.1

- 基本按鍵編輯與顯示
- 懸浮窗顯示
- KPS / TOTAL 統計
- 配置保存

### v1.2

本次整理為目前程式碼實際狀態，並加入今天已完成的修正：

- 修正懸浮窗背景透明，避免把編輯畫布背景一起畫進懸浮窗
- 修正深色 / 淺色切換時沒有實際動畫的問題
- 修正全局設置面板的輸入框在深淺色切換時顏色不更新的問題
- 修正顏色設置面板展開 / 收起動畫，避免像遮罩蓋住上方內容
- 修正底部「切換輸入法」按鈕尺寸與其他兩顆按鈕不一致的問題
- 修正批量選取後無法整組拖動的問題
- 修正圖層管理中的失效點：
  - `bringToFront()` 現在會真正更新 `zIndex`
  - 重設圖層時不再全部歸零，而是重新分配穩定層級
  - 手動修改 `zIndex` 後會做正規化，避免層級混亂
- 修正 KPS 統計不再被固定長度陣列限制
- 修正按鍵統計保存策略，改為批次保存與定時保存，降低高 KPS 下的寫入負擔
- 關閉懸浮窗與 Activity 結束時，會補做一次統計資料 flush
- 修正多個輸入裝置只監聽到第一個 device 的問題，現在會並行監聽多個 keyboard event device
- 加入多鍵同時按下的診斷 log，方便分析 6KRO / 7 鍵上限或鍵盤 rollover 問題
- 修正配置導出可能只建立空檔案但內容寫不進去的問題，改善 `OutputStream` fallback 寫法

## 已知限制

- 某些鍵盤本身只有 `6KRO` 或類似 rollover 限制，這種情況下 App 端無法憑空補出硬體沒有送出的按鍵事件
- 專案目前仍依賴 `Shizuku`；若沒有可用的 Shizuku 環境，按鍵監聽無法正常工作
- 按鍵映射尚未完成，不應把 README 視為映射功能完成公告

## 使用方式

1. 安裝並啟動 `Shizuku`
2. 安裝並開啟 KeyViewer
3. 在主畫面授權 Shizuku
4. 新增按鍵並綁定要顯示的鍵
5. 依需求調整位置、大小、顏色、圖層與鍵雨效果
6. 如需懸浮顯示，啟用懸浮窗權限並開啟懸浮窗
7. 可用配置匯入 / 匯出保存版面
